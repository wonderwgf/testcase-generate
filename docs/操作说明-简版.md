# 测试用例生成工具 · 快速操作手册

---

## 一、团队共享（克隆仓库方式）

### 初次配置（每人只做一次）

**第一步：克隆工具仓库**

```bash
git clone <仓库地址> testcase-generate
```

**第二步：用 Cursor 打开仓库根目录**

```
File → Open Folder → 选择 testcase-generate/
```

> 打开后 `.cursor/commands/` 和 `.cursor/skills/` 自动生效，`/testcasegen` 命令立即可用。

**第三步（可选）：将项目数据目录移出工具仓库**

工具仓库本身不存放业务数据。建议每个项目在**其他位置**单独建目录，通过 Cursor 多工作区或软链接方式管理。也可以直接在工具仓库根下新建项目目录（如 `acflow/`），用 `.gitignore` 排除数据文件。

---

### 日常使用

工具仓库不需要每次打开，**只需保证 Cursor 工作区根目录下能找到 `.cursor/commands/`** 即可。

- 如果项目数据就在工具仓库内：直接在工具仓库目录下使用
- 如果项目数据在其他目录：将工具仓库根目录加入 Cursor 工作区（`File → Add Folder to Workspace`）

---

### 获取工具更新

```bash
cd testcase-generate
git pull
```

拉取后重启 Cursor（或重新加载窗口）即可生效。

## 二、第一次用（全新项目）

```
第一步：初始化项目目录
  /testcasegen <项目> <迭代> 初始化

第二步：放入需求文档和知识库
  <项目>/input/prd/<迭代>/      放本迭代需求文档（.docx / .xlsx）
  <项目>/input/knowledge/...    放知识库文档（.md 格式，可选）

第三步：运行命令
  /testcasegen <项目> <迭代>

第四步：跟着 Agent 走
  有风险点 → Agent 暂停，确认后说「继续」
  全部完成 → 自动导出 XMind
```

**示例**：`/testcasegen acflow V4.5`

---

## 三、目录结构（只需关注 input 部分）

```
<项目>/
├── input/
│   ├── prd/
│   │   ├── V4.5/          ← 本迭代需求文档（docx/xlsx）放这里
│   │   └── 通用规则.xlsx   ← 跨迭代共用的文档放根下（不在子目录中）
│   └── knowledge/         ← 知识库（md格式，跨迭代共享）
│       ├── baseline_cases/ ← 历史测试用例
│       ├── business/       ← 业务规则文档
│       ├── codedesign/     ← 技术设计文档
│       └── prd/            ← 历史需求文档
└── output/                ← 所有产出自动生成，无需手动创建
```

## 四、常用命令

| 想做什么 | 命令 |
|---------|------|
| 全流程（最常用） | `/testcasegen acflow V4.5` |
| 只转换文档格式 | `/testcasegen acflow V4.5 只文档转换` |
| 从需求解析开始 | `/testcasegen acflow V4.5 需求解析` |
| 从测试概要开始 | `/testcasegen acflow V4.5 测试概要` |
| 从测试用例开始 | `/testcasegen acflow V4.5 生成用例` |
| 只导出 XMind | `/testcasegen acflow V4.5 导出` |
| 只刷新知识库索引 | `/testcasegen acflow 只刷索引` |

## 五、生成流程一览

```
Step 1   初始化目录          自动
Step 2   文档转换（docx→md） 自动
Step 2.5 知识库索引          自动
Step 2.7 大文档拆分检测      自动（>800行时触发，需确认方案）
──────────────────────────────────────
Step 3   需求解析            AI生成 → 有风险点时暂停
Step 4   测试概要            AI生成 → 有缺口时暂停
Step 5   测试用例            AI生成 → 大文档预算触顶时暂停
──────────────────────────────────────
Step 6   合并 + 导出XMind    自动
```

**暂停时**：Agent 会列出原因，说「继续」即可进入下一步。

---

## 六、大文档怎么办

需求文档 md 超过 **800 行**时自动进入模块循环模式：

1. Agent 展示拆分方案 → 说「确认」
2. 逐模块处理，每步有连续处理上限（4~6 个/步）
3. 触顶时提示「新开会话继续」，重新运行相同命令即可自动续接

---

## 七、知识库怎么维护

- 文件必须是 **md 格式**；若是 docx/xlsx，先运行 `/testcasegen <项目> 只文档转换`
- 放入对应子目录后，知识库索引会在下次运行时自动刷新
- 知识库跨所有迭代共享，放一次长期有效

---

## 八、中途退出后怎么继续

直接重新运行原命令：

```
/testcasegen acflow V4.5
```

Agent 自动检测已完成的步骤，从上次中断处继续，无需任何额外操作。
