# 测试用例生成规则（统一）

**适用范围**：由产品需求（PRD/需求规格说明书等）生成测试用例文档时，按本规则统一结构、拆条粒度和书写格式，便于评审、执行与维护。

---

## 一、文档与层级约定

### 1.1 文档头部

- 第一行为**主题标题**：`# 模块名-测试用例` 或 `# 模块名-测试用例（版本/来源说明）`。
- 紧接着写 **来源、生成方式、产出日期**（可 2～3 行），再 `---` 分隔正文。

示例：

```markdown
# XX模块-测试用例

**来源**：PRD 文档名或需求章节。
**生成方式**：按 PRD 第 X 章/接口文档逐条转化为测试用例。
**产出日期**：YYYY-MM-DD

---
```

### 1.2 标题层级与含义

| 层级 | 写法 | 含义 | 示例 |
|------|------|------|------|
| 一级 | `#` | 文档主题 | 交单-测试用例 |
| 二级 | `##` | 端/大模块/大章节 | 内管端-平台审核列表页、客户端-XX列表、XX接口对接 |
| 三级 | `###` | 页面/tab 组/步骤组 | 待领取、交单信息 tab、请求参数 |
| 四级 | `####` | 子模块/筛选项组/列表字段组 | 查询筛选项、列表字段 |
| 五级 | `#####` | **单条用例标题**（可执行用例） | 业务编号、索款金额-校验≤可用余额且≤单据金额 |

**原则**：  
- **一条可执行用例 = 一个 ##### 标题 + 下面的四要素（前置、操作、预期、优先级）**。  
- 仅作分组的标题（如「查询筛选项」「列表字段」）下不直接写四要素，其下每个 ##### 才是一条用例。

---

## 二、单条用例格式（四要素）

每条可执行用例**必须**包含且仅包含以下四项，顺序固定：

```markdown
##### 用例标题（一句话概括场景或校验点）
- 前置：执行本用例前需满足的条件（角色、权限、数据、配置、当前页面/tab 等）
- 操作：用户或系统执行的动作（步骤可 1～3 步，用分号或序号分隔）
- 预期：应出现的结果（展示、校验、状态、落库、接口调用等，可多条）
- 优先级：P1 / P2 / P3
```

**书写要求**：  
- **前置**：写清「谁、在什么环境下、当前处于什么页面/状态」；若有配置项或数据条件，写具体（如「列表有至少一条数据」「配置项 XX 为 是」）。  
- **操作**：动词开头（进入、点击、输入、选择、提交、查看、调用…）；控件名用【】如【提交】；输入类型可括号说明（文本模糊搜索、区间选择器、下拉单选 枚举值：…）。  
- **操作 / 预期编号对齐（合并规则）**：同一条用例中，若「操作」「预期」均使用 **`- 操作：` / `- 预期：` + 编号列表（`1.` `2.` …）** 书写，则：**当操作下编号步骤 ≥2 条、且预期下编号步骤仅 1 条时**，须将操作**合并为 1 条**，各步用 **中文分号「；」** 串成一句（仍保留 `  1. ` 起头）；**当预期下编号步骤 ≥2 条、且操作下编号步骤仅 1 条时**，须将预期**合并为 1 条**，各项同样用 **中文分号「；」** 串成一句（仍保留 `  1. ` 起头）。**不合并**的情形：操作与预期编号条数**同为 1** 或**同为 2 条及以上**；或任一侧为**单行叙述无编号列表**。细则与目的见 **§8.8**。  
- **预期**：可验证、无歧义；涉及校验时写出**完整提示文案**；涉及状态/落库时写出字段名或状态枚举值。  
- **优先级**：P1 核心流程/主路径，P2 常用分支/列表展示/配置，P3 边界/异常/低频。

## 三、拆条规则（何时一条、何时多条）

### 3.1 一个字段/一个控件一条用例

**适用**：列表筛选项、列表列、表单展示/编辑字段、接口请求/返回的单个参数。

- **列表-查询筛选项**：每个筛选项一条用例。  
  - 前置：该列表/tab 有数据（或「有至少一条数据」）。  
  - 操作：输入/选择该筛选项（注明类型：文本模糊、区间、下拉单选 枚举…）。  
  - 预期：筛选生效，列表结果与条件一致（或按需求写具体预期）。  
  - 优先级：一般 P2。

- **列表-列表字段**：每个表头/列一条用例。  
  - 前置：该列表/tab 有至少一条数据。  
  - 操作：查看列表。  
  - 预期：列表展示 XX 列（写清列名）。  
  - 优先级：一般 P2。

- **表单/详情-展示或编辑字段**：每个业务字段或控件一条用例。  
  - 前置：进入对应页面/tab，必要时写明数据或配置条件。  
  - 操作：查看该字段/控件（或修改后失焦、提交）。  
  - 预期：展示规则、可编辑与否、计算公式、校验提示（写完整文案）。  
  - 优先级：主字段 P1，次要 P2。

- **接口-请求/返回参数**：需要逐项验证时，每个参数一条用例。  
  - 前置：触发该接口的场景（如「提交交单」「定时任务调用来单通知列表」）。  
  - 操作：传参/解析该参数。  
  - 预期：中文名称、是否必输、取值来源/落库规则、格式要求。  
  - 优先级：必输项 P2，选填 P3。

### 3.2 一个业务分支一条用例

**适用**：同一功能因配置、枚举、接口返回值等不同而有不同结果时，按**分支维度**拆成多条。

- **配置项分支**：每个配置取值一条（或「已配置/未配置」各一条）。  
  例：未配置企业交单审核时提交→弹出签署；已配置企业交单审核时提交→状态为交单企业审核。

- **枚举/类型分支**：每个枚举值或类型一条（若数量很多可合并为「枚举值符合需求文档」一条，或抽样）。  
  例：付款期限为即期/运输单据装运日后/见单日后… 分别写福费廷到期日计算。

- **接口/返回值分支**：按返回码、状态码、关键字段取值拆条。  
  例：orderState=91 时业务处理、orderState=94/99 时业务处理；dscDesc 为空/不为空 × 承付方式 × 是否配置自定义承付子流程 各一条。

- **审批/流程节点分支**：按节点 + 结论拆条。  
  - 每个审批节点一个 ####，其下一级（##### 用例标题）**仅展示结论**：「通过」「拒绝」；若该节点支持退回，再增加「退回」。**不重复节点名**（如不用「业务经办审批-通过」，而用「通过」）。  
  例：#### 业务经办审批 → ##### 通过、##### 拒绝；#### 业务部门负责人审批 → ##### 通过、##### 退回、##### 拒绝。

- **并行分支 / 并行网关（与串行节点并列，勿与单节点「通过/拒绝」混为一谈）**：PRD 或流程图中出现**并行分叉**（多路径同时执行后**汇聚**）或**并行审批**（或签、会签、多实例等）时，除串行节点用例外，需按下列补条（与 PRD 用语一致）：  
  - **并行路径**：为**每条并行分支**至少一条主路径用例（完成该分支上的关键动作）；在**汇聚点**至少一条（全部满足汇聚条件后的状态/接口/待办；若 PRD 规定「任一分支异常则整体失败」则另写对应分支）。  
  - **并行审批**：按 PRD 拆——**会签**常见组合如「全部通过」「一人拒绝」「部分通过未汇聚」等；**或签**常见组合如「任一通过即过」「全部拒绝才失败」等；**串并行混合**时按**阶段**先分块，再在每块内套用串行「节点+结论」或本款并行规则。  
  - **层级**：可与串行同级用 `#### 并行分支名称` / `#### 会签-XX节点`，其下 `#####` 仍为单条可执行用例；避免整条流程只写「并行」一条带过。

**规则**：若两个维度组合（如 A×B），且组合后行为不同，则**每个组合一条用例**；若行为相同可合并为一条并前置中写明「A 或 B」。

### 3.3 一个操作/一个按钮一条用例

**适用**：列表或页面的每个可点击按钮、每个主要操作。

- 前置：当前页面/列表状态、是否有数据、是否已勾选等。
- 操作：点击【按钮名】或执行操作（必要时写二次确认）。
- 预期：跳转、弹窗、状态变更、接口调用、数据退回任务池等。
- 优先级：主流程按钮 P1，辅助 P2。

### 3.4 不拆条、一条总述的情况

- **数据范围与排序**：如「待领取 仅展示未领取待办且按创建时间倒序」一条即可，不必再按「未领取」「创建时间倒序」拆两条。
- **整块展示/布局**：如「详情页左右分栏与交单信息、信用证信息」一条，再在下一级按 tab 或区块拆字段级用例。
- **接口总述**：如「请求体包含接口文档规定的全部请求参数；必输项有值且符合约定」可作为一条总用例，再视需要补充参数级用例。

## 四、按页面类型的结构顺序

### 4.1 列表页（含多 tab）

1. **菜单路径/入口**（可选，一段说明）。  
2. **按 tab 分**：每个 tab 一个 ###。  
3. **每个 tab 内顺序**：  
   - 数据范围与排序（一条 #####）。  
   - #### 查询筛选项 → 其下每个筛选项一条 #####。  
   - 若采用「**`#### 筛选项` + 每字段 `#####` + 场景 `######`」**：在「数据范围与排序」**`#####` 用例**之后增加 **`#### 筛选项`**（下可接 `>` 说明）；其下**每个查询条件字段一节 `#####`**（标题仅用字段/控件名，**勿**用括号备注控件类型），**每条场景 `######`**。与「仅单条数据范围」类 **`#####` 用例**的区分：**筛选项分组内**的 **`#####`** 必须以 **`######` 场景**为下一非空标题（否则视为普通 `#####` 单条用例）。**`#### 【查询】` / `#### 【重置】` / `#### 【导出】`**、**`#### 列表字段`** 等与「筛选项」**并列**（同级 `####`）；常见顺序：数据范围 → 筛选项（`#####` 字段群）→ 列表字段 → 行内操作 → 工具栏按钮 → 其他。  
   - **简版/旧版**：无 **`#### 筛选项`** 时，仍可「**每个筛选项独立 `####`**、场景 `######`」；标题规则同上（勿括号备注类型）。  
   - #### 列表字段 → 其下每列一条 #####。  
   - **列表工具栏按钮（查询 / 重置 / 导出等）**：**一个按钮一个 `####` 主题**（标题写 `#### 【查询】`、`#### 【重置】`、`#### 【导出】` 等），**不要**混在「#### 按钮与操作（查询·重置·导出）」一条里；**该按钮下的各条测试场景**用 `######` 挂在对应按钮标题下（如 `###### 已改条件后点击刷新列表`、`###### 清空全部筛选条件（多项同时有值）`）；**【重置】** 下场景名**不再**加「重置-」前缀（已在 `#### 【重置】` 下区分）；**【导出】** 下宜按筛选项拆 **N 条「单条件过滤后导出」+ 1 条「多条件组合后导出」**，场景名**不再**加「导出-」前缀（详见 **§8.6**）。  
   - **工具栏与筛选项：同级且同等重要**：`#### 查询筛选项`（或 `#### 筛选项` 及下属字段群）与 `#### 【查询】`、`#### 【重置】`、`#### 【导出】` 均为同级 `####`，**覆盖优先级相同**。**禁止**仅用一条概括【重置】/【导出】充当「占位」、或以「与筛选项重复」为由删减；**【重置】须满足 §8.5 的 1+N、【导出】须满足 §8.6 的 N+1**（及按需 **§8.7**），并与当前 tab 筛选项枚举的 **N** 一致（**§8.0**）。  
   - **其他行内/批量操作**（批量领取、领取、办理、退领、流程追踪等）：仍按 **一个操作一条 #####**（或按产品分组为 `####` + `######`）。

### 4.2 表单页/申请页/详情页

1. **菜单路径/入口**（可选）。  
2. **页面结构/步骤**：总述（如三步流程、左右分栏）一条 #####。  
3. **按步骤或 tab**：每个步骤/tab 一个 ###，其下按**字段/控件**或**区块**拆 #####；有分支的（如按配置、按枚举）在该区块下再拆 #####。  
   - **`####` 与下属 `#####` 归属与不重复父名（建议）**：  
     - **归属**：仅描述某一表单区块的用例，应放在对应 `####` 下；**不要**在该 `####` 下写专属于其它区块、且前置/操作又未体现本区块语境的用例。  
     - **标题不重复父级主题词**：父级 `####` 已写明区块（如「上传资料-贸易发票」）时，其下 `#####` **不必在标题中再写**「贸易发票」等与父级重复的字样，以**字段名、按钮名或场景结论**作标题即可（与 **§3.2** 审批里「#### 业务经办审批 → ##### 通过/**不重复节点名**」同一原则）。若标题过短可能与其它 `####` 混淆，可在**前置首句**用「本区块列表」「本区」等点明，而**非**必须重复父级全称。  
     - **整页按钮**：【提交】等若因**本区块规则**触发校验，在**操作/预期**中写清触发条件即可（例：「列表无数据行时点击页底【提交】」——该条已挂在「上传资料-贸易发票」下则不必标题再写「贸易发票」）。  
4. **按钮与提交**：取消、暂存、提交及各提交分支（按配置或结果不同）各一条 #####。

### 4.3 接口/对接章节

1. **接口区分**：哪个场景调哪个接口，一条 #####。  
2. **每个接口下**：  
   - 调用场景（可选）一条。  
   - #### 请求参数：总条（可选）+ 按需参数级 #####。  
   - #### 返回参数：总条（可选）+ 按需参数级 #####。  
3. **业务处理/分支**：按返回码、关键字段取值、配置组合拆 #####（见 3.2）。

### 4.4 定时任务/后台逻辑

1. 触发条件与执行顺序（一条 #####）。  
2. 调用的接口及请求/返回（可引用接口章节或本条下子条）。  
3. 按返回结果或业务分支拆 #####（成功/失败、有数据/无数据、配置 A/配置 B 等）。

## 五、优先级统一口径

| 优先级 | 含义 | 典型场景 |
|--------|------|----------|
| **P1** | 核心流程、主路径、必验功能 | 主流程提交、主列表展示与操作、关键校验（如索款金额≤可用余额）、关键接口调用、状态正确更新 |
| **P2** | 常用分支、配置、展示、次要接口 | 配置项开关、tab 切换、筛选项与列表列、详情展示与只读字段、接口必输参数、审批分支 |
| **P3** | 边界、异常、低频、补充验证 | 边界值、异常提示、低频配置、接口选填参数、兼容/历史逻辑 |

## 六、禁止与建议

### 6.1 接口对接类用例（拆条）

#### 禁止

- 接口对接类用例中，将同一接口的多个请求字段或多个返回字段合并在一条用例里做断言（允许的「接口总述」单条除外，总述后仍须按字段逐条补全）。
- **禁止**生成的测试用例与接口文档**字段不对应**：用例所覆盖的请求/返回字段须与接口文档**字段表**在名称、含义、层级上可一一对应；**不得**使用接口文档未定义的字段名，**不得**遗漏接口文档已列出字段而不写对应用例（必输项与选填项均须按接口表逐字段覆盖，选填无值时预期内容与操作内容一致，仍单列）。

### 6.2 通用禁止

#### 禁止

- 一条用例包含多个互不相关的校验点（应拆成多条）。  
- 预期写「正常」「符合需求」等不可验证的表述。  
- 前置缺失或与操作不匹配（如前置写「未配置」却操作「提交后平台审核」）。  
- 用例标题与四要素不一致（标题写 A，预期写 B）。  
- **场景概括（`#####` 标题）中禁止**堆砌、复述已由上级 `#`～`####` 标明的**端、模块、菜单、页面全路径**及与父级**重复的主题词**（标题宜短，以字段名、按钮名、条件与结果为主）。**操作**、**预期**不受本条限制，可写明页面/区块/字段全名、模块、路径、完整提示文案等，便于执行与核对。

### 6.3 建议

#### 建议事项

- **`####` 子级归属与不重复父名**：见 **§4.2** 第 3 款；父级已写区块名时下属 `#####` 标题勿再叠床架屋重复同一主题词（同 **§3.2**「不重复节点名」）。  
- 用例标题用**动宾或「场景-结果」**，如「索款金额大于可用余额时红字提示且不可提交」。  
- 同一模块内前置可复用时可简写，如「在 XX 列表-待领取 tab」；首次出现时写全。  
- 涉及枚举的，在操作或预期中写明枚举值。

## 七、模板速查

**列表筛选项单条**：

```markdown
##### 字段名
- 前置：XX 列表（或 XX tab）有数据
- 操作：输入/选择 字段名（类型说明，如文本模糊搜索、区间选择器）查询
- 预期：筛选生效，列表结果与条件一致
- 优先级：P2
```

**列表列单条**：

```markdown
##### 列名
- 前置：XX 列表（或 XX tab）有至少一条数据
- 操作：查看列表
- 预期：列表展示 列名 列
- 优先级：P2
```

**表单字段/控件单条**：

```markdown
##### 字段名或控件名
- 前置：在 XX 页面/tab，可选数据或配置条件
- 操作：查看/输入/选择 该字段或控件
- 预期：展示规则或可编辑性；若有校验写完整提示文案
```

## 八、界面筛选项与列表：场景检查清单

### 8.0 工具栏用例的生成顺序（重置 / 导出）

1. **同级、同等重要**：列表页中，**查询筛选项**与工具栏 **`#### 【查询】`、`#### 【重置】`、`#### 【导出】`** 均为同级 `####`，与 **§4.1**「工具栏与筛选项：同级且同等重要」一致；评审时与筛选项块**同一标准**，不得降级为「可选项」或单条占位。  
2. **先查筛选项，再写工具栏**：编写 **`#### 【重置】`、`#### 【导出】`** 前，须先在当前列表/tab 的 **「查询筛选项」**（及 PRD 列表区）中**枚举全部独立筛选项控件**，确定 **N**（每个可单独输入/选择的查询条件计 1；同一物理字段只计 1；日期区间、金额区间各计 1）。  
3. **再按 N 落条数**：**§8.5** 按 **1 + N** 条写【重置】（1 条多项同时有值 + N 条仅单筛选项有值）；**§8.6** 按 **N + 1** 条写【导出】（N 条单筛选项过滤后导出 + 1 条多条件组合导出）；需要时按 **§8.7** 再增「无筛选/默认列表导出」。**两处所用 N 须同源、一致**。  
4. **【查询】**：仍用独立 `#### 【查询】`；场景条数按产品需要，但至少保留「改条件后刷新」类可执行场景，**不得**与【重置】【导出】混写在同一 `####` 下。

生成或评审列表页用例时，除「逐字段各一条」外，宜按下表**按类型补全场景**（与 PRD/产品实现一致处写死预期，不一致处标【待确定】）。

### 8.1 筛选项（按控件类型）

#### 结构与层级

- **筛选项字段单独作为一个主题**：列表页筛选项推荐 **`#### 筛选项`** 下挂 **`#####` 字段节**（每查询条件一节）；无「筛选项」总节时，字段仍可用 **`####`**（与 §4.1「简版/旧版」一致）。字段标题**勿**用括号备注控件类型。  
- **可选分组标题 `#### 筛选项`**：需要时在字段群前增加 **`#### 筛选项`**（含简要 `>` 说明即可）；其下查询字段用 **`#####`**，**勿**与「筛选项」同级仍用 **`####`** 写字段名（以免与列表字段等同级 `####` 混淆）；**`#####` 的下一非空标题须为 `######` 场景**（否则按普通单条 `#####` 用例解析，见 §4.1）。  
- **字段的下一级是场景**：该字段节下不再把「同字段多场景」挤在同一条用例里，应**按场景拆条**；每条可执行场景使用 **`###### 场景标题`**，其下写前置、操作、预期、优先级（**格式同 §2 四要素**；列表筛选项详版采用六级标题区分于简版 `#####`）。  
- **与下表关系**：下表按**控件类型**列出宜覆盖的场景类型，落笔时归入对应字段的 `######` 下，避免遗漏。

| 类型 | 建议覆盖的场景 |
|------|----------------|
| **文本 / 编号模糊** | 与产品约定一致的多种输入形态（如编号：空格+关键字、分段、尾空格、特殊字符、仅关键字等）；名称类可与编号侧**同构对照**时，场景标题可用字面——**名称类包含人名和企业名**：人名场景标题可用字面（如「张三」），企业名场景标题可用字面（如「上海」）；**编号类与名称类标题勿混用**。 |
| **日期区间** | 同一天；跨日典型组合（如昨天～今天、今天～明天，以 PRD 为准）；**不填**该区间；支持时**精确到日/单点**查询。 |
| **金额及利率、利息等数值区间** | 起止同值（如均 100）；跨段（如 100～200）；含 0；负数与上界组合（如 -1～200）及**禁止负**时的提示/拦截；**不填**区间；**等值/精确**查询（若产品支持）；**精度展示**（如输入 `100.001` 后输入框展示 `100.00`，与 DECIMAL 规则一致，避免 FLOAT 口径）。 |
| **枚举（下拉）** | 下拉项与 PRD 一致；**各枚举值选择后查询**；**不选择**（仅该控件未选/清空后查询）。 |
| **全页通用** | **不填写**任一筛选项直接查询；**精确查询**（字段支持全等时，与列表行完全一致）。 |

### 8.2 列表

| 维度 | 建议覆盖的场景 |
|------|----------------|
| **列与数据** | 表头、列顺序、合并表头下的各业务列均有验证；格式、单位、精度、空值/占位符、与接口或 PRD 字段映射。 |
| **与筛选联动** | 筛选后结果在对应列可核对；重置/清空后列表恢复默认范围（若 PRD 有约定）。 |
| **行为** | 排序、分页（若 PRD 有）；合计/小计（若有）；默认条数与翻页边界。 |
| **操作列 / 按钮** | **列表工具栏**：`#### 【查询】`、`#### 【重置】`、`#### 【导出】` **各一主题**，其下用 `######` 写该按钮专属场景（勿混在「按钮与操作」一条里）；**行内/批量按钮**：**显隐、权限、与业务状态联动**（按 PRD）。 |

### 8.3 通用要求（与第三章呼应）

- **逐字段、禁止抽样**：筛选项与列表列能枚举的尽量**逐条**落用例，不用「筛选项正确」「列表展示正确」一条带过。  
- **PRD 直出模式**：先建立**列表要素覆盖清单**，再保证清单**逐行有对应用例**。  
- **与「零-B」关系**：详见 `03testrequirement.mdc` 中 B1/B2/B3。

### 8.4 易混淆概念（避免重复或写错）

| 概念 | 含义 |
|------|------|
| **整页不填写** | 所有筛选项不填或保持默认空 → 点击【查询】。 |
| **枚举不选择** | **仅某一枚举控件**未选或清空 →【查询】；与整页「不填写」不同。 |
| **区间不填写** | **仅该日期/金额区间控件**不填 →【查询】。 |
| **精确查询** | 对支持全等的字段，取值与列表某行**完全一致**后查询。 |

### 8.5 重置按钮（【重置】）用例编写规则

**编写前提**：已按 **§8.0** 枚举本列表 **N 个独立筛选项**；**先生成或核对「查询筛选项」再写【重置】**，保证下文第 4 款「N 条」与筛选项一一对应，**禁止**在未确定 N 时只写 1 条概括。

1. **结构位置**：列表页将【重置】列为独立小节，使用 **`#### 【重置】`**；其下每条可执行场景用 **`######`**，**勿**与【查询】【导出】混写在一个「按钮与操作」总条里。  
2. **场景标题命名**：因已挂在 **`#### 【重置】`** 下，**不再**在标题文案前加「重置-」；直接写场景含义，例如 **`###### 清空全部筛选条件（多项同时有值）`**、**`###### 仅【信用证编号】有输入后点击`**。简版若用 **`##### 验证…`**，写 **`验证清空全部筛选条件（多项同时有值）`**、**`验证仅信用证编号有输入后点击`**（同样不加「重置-」）。  
3. **必备总场景（1 条）**：已在**多个**筛选项中同时输入/选择条件 → 点击【重置】→ 预期**全部筛选项清空**（或恢复页面默认态）、**列表恢复默认范围/全量**（与 PRD 一致）。  
4. **按筛选项拆分（N 条）**：当前列表区有 **N 个独立筛选项控件**，则宜再写 **N 条**：每一用例仅让**某一个**【筛选项】有值（其余均不填、不选），再点【重置】；预期该字段被清空、列表恢复默认，并写明**其它筛选项无异常残留**（若产品为「一键清空全部条件」则与本条等价，可在预期中写「仅该字段曾赋值，重置后与其它项一致」）。  
5. **控件类型差异**：文本/编号类写「输入关键字」；枚举写「选择一项」；日期区间写「选择起止」；金额区间写「填写最小/最大之一或区间」——**操作描述与真实控件一致**。  
6. **前置**：写明所在菜单/列表 tab（如【交单管理】-【待操作】），保证可执行。  
7. **与「整页不填写」区别**：「不填写」是**不点重置**、直接【查询】的通用场景；「重置」是**点击【重置】**后清空条件，二者都要覆盖，**不得互相替代**。  
8. **优先级**：一般为 **P2**（与筛选项类一致）。

**生成测试用例时的速记**：`#### 【重置】` → 1 条「多项同时有值」+ N 条「仅单筛选项有值」；标题**无「重置-」前缀**；详版 `######`，简版 `##### 验证…`。

### 8.6 导出按钮（【导出】）用例编写规则

**编写前提**：**N** 与 **§8.0、§8.5** 所用独立筛选项个数**一致**；**先枚举筛选项再写【导出】**，按第 3～4 款写满 **N + 1** 条（另按需 **§8.7**），**禁止**用单条「导出当前结果」替代。

1. **结构位置**：列表页将【导出】列为独立小节，使用 **`#### 【导出】`**；其下每条可执行场景用 **`######`**，**勿**与【查询】【重置】混写在一个「按钮与操作」总条里。  
2. **场景标题命名**：因已挂在 **`#### 【导出】`** 下，**不再**在标题文案前加「导出-」；直接写场景含义，例如 **`###### 仅【信用证编号】筛选后导出`**、**`###### 多项筛选条件组合后导出`**。简版可用 **`##### 验证仅信用证编号筛选后导出`**、**`##### 验证多项条件组合后导出`**。  
3. **按筛选项拆分（N 条）**：当前列表区有 **N 个独立筛选项**，则宜 **N 条**：每一用例**仅使用其中一个筛选项**设置过滤条件（其余不填不选）→ 按产品要求执行 **【查询】**（若必须先查询再导出，操作链中写明）→ 点击 **【导出】**；预期导出内容与**当前筛选结果**一致（条数、字段、与列表可核对；文件格式/命名以 PRD 为准）。  
4. **组合场景（1 条，必备）**：已在 **多个**筛选项中同时设置条件 → 【查询】→ 点击【导出】；预期导出范围与**组合条件下列表结果**一致（或 PRD 约定的导出口径，如是否含合计行）。  
5. **可选补充**：若 PRD 要求验证「无任何筛选直接导出」「默认条件导出」，可各增一条，**不替代**上述 **N+1** 结构。  
6. **优先级**：一般为 **P2**。

**生成测试用例时的速记**：`#### 【导出】` → **N** 条「每个筛选项单独过滤后导出」+ **1** 条「多筛选项组合后再导出」；标题**无「导出-」前缀**；操作链一般为 **设条件 → 查询 → 导出**（以产品交互为准）。

### 8.7 无筛选 / 默认条件导出（追加说明）

本条为 **§8.6 的补充**，**不替代、不修改** §8.6 已有各款。

- **适用**：需验证「未修改任何筛选项即导出」或「默认列表口径下导出」时，在 **`#### 【导出】`** 下**再增一条**，与 §8.6 第 3～5 条**并列**。  
- **标题示例**：`###### 无筛选条件时导出（与当前默认列表一致）`。  
- **操作**：筛选项保持默认空/未改 → 按需【查询】→【导出】（是否必须先点【查询】以产品为准）。  
- **预期**：导出范围与页面**当前默认列表**或 PRD 约定口径一致。  
- **优先级**：一般为 **P2**。

**速记补充**：在 **N+1** 基础上，需要时 **+1** 条「无筛选导出」。

### 8.8 操作与预期编号条数对齐（与第二章「书写要求」一致）

与 **第二章「四要素」→ 书写要求** 中「操作 / 预期编号对齐（合并规则）」**同一条款**，此处作展开备忘。

- **何时合并（操作侧）**：`- 操作：` 下**带编号步骤 ≥2 条**，且 `- 预期：` 下**带编号步骤仅 1 条** → 将「操作」**合并为 1 条**，各步用 **中文分号「；」** 连接（仍保留首部 `  1. `）。  
- **何时合并（预期侧）**：`- 预期：` 下**带编号步骤 ≥2 条**，且 `- 操作：` 下**带编号步骤仅 1 条** → 将「预期」**合并为 1 条**，各项用 **中文分号「；」** 连接（仍保留首部 `  1. `）。  
- **何时不合并**：操作与预期**同为 1 条编号**或**同为 2 条及以上编号**；或任一侧为**单行叙述无编号列表**。  
- **目的**：避免「一侧多步、另一侧单条总判」导致层级不对称；合并后仍保持单条可执行、可验证链。

### 8.9 新增界面

> 本小节为**新增/录入类界面**（如「新增交单申请」）的补充口径，来源于已落地用例实践；**不替代**前文规则。

#### 8.9.1 界面定位与文档入口

- **新增界面**：指用户**新建一条业务**的录入页，与列表页、详情只读页区分。
- 在 `###` 小节下建议先用 **`> **页面元素**`**（或等价说明）固定**区块划分、控件类型、区上红字/业务规则**，后续用例的「前置/预期」可引用该块，避免重复贴 PRD 全文。

#### 8.9.2 表单区标题层级（模块 → 字段 → 场景）

- 与**页面大标题/模块**对齐：例如界面大标题「交单承付及福费廷信息」作 **`####` 模块**（若界面拆成多个子标题，可合并为一个 `####` 或按子标题拆分，以产品为准）。
- **`#####` = 字段**（单控件/单行表单项）。
- **`######` = 具体场景**（该字段下的校验、联动、只读等），每条仍须包含**前置 / 操作 / 预期 / 优先级**（与第二章「四要素」一致）。
- **归类用语**：**金额类**、**企业名称类**写在字段说明中即可；**利率类**（如「福费廷利率」）按**金额类同口径**编写（小数位、非法字符、负数、上限等），避免与金额校验口径脱节。

#### 8.9.3 表格区（上传资料类列表）

- **表格内列字段不参与**「模块 → 字段 → ######」这一套层级；**不按列再拆 `######`**。
- 每个表格一个小节 **`#### 上传资料-xxx`**（或产品等价命名），建议结构为：
  - 先写 **`**列表展示列**`**（列名、操作列按钮、表底合计等），区上红字/规则**指回**本节或「页面元素」引用。
  - **「查看列表」类用例**：标题宜为 **「查看 xxx 列表展示字段」**；**操作**为查看表头/列表；**预期只写列表展示的字段**（与本节「列表展示列」一致），不在此条展开「有附件时可点」等交互细节。
  - 其余**简单场景**（必传拦截、金额与融资包关系、添加行、附件大小、分页等）仍用 **`#####`**，与 §8.9.2 的表单层级脱钩，按业务点逐条列示。
  - **更细的「新增合同/新增发票」等行内用例**可**后续补充**，不阻塞主结构。

#### 8.9.4 整单级操作

- **【提交】【保存】及底部操作** 单独 **`#### 提交（整单）` / `#### 保存（整单）` / `#### 底部操作`**（命名可与项目统一）。
- 其下用 **`#####`** 表示**整单维度**场景：全量必填成功、某必填项缺失拦截、与详情/再进入一致性等。
- 若 PRD 规定【保存】校验弱于【提交】，相应用例标【待确定】并按 PRD 调整预期。

#### 8.9.5 页面级分区浏览（可选）

- 可用 **`#####`** 描述**分区是否可见**、**从列表进入查看**等；预期描述**主区块标题与页底按钮**；与 §8.9.3「表格仅列字段」的颗粒度区分清楚。

#### 8.9.6 与列表页用例的边界

- **通用筛选、列表字段、导出**等仍按第四章等**列表页**规则编写。
- **新增界面**重点在：**录入结构、表格列说明、表单字段校验、整单提交/保存**；勿与列表页筛选项场景混写于同一条预期中。

| **列表页结构** | **§4.1**：数据范围 → 筛选项 → 列表字段 → 行内操作 → **`#### 【查询】`【重置】【导出】**；**筛选项与三按钮同级、同等重要**。 |
| **重置 / 导出** | **§8.0**：**先**枚举本 tab 独立筛选项得 **N**，**再**写【重置】**1+N**、**【导出】N+1**（+按需 **§8.7**）；N 在三处一致。 |
| **清单与场景** | **§8.3**：逐字段、宜建列表要素清单；**§8.1～8.2** 按控件类型与列表维度补场景；**§8.8**：操作/预期一侧多步另一侧单条时做合并对齐。 |
| **易混** | **§8.4**：整页不填、枚举不选、区间不填、精确查询、重置与整页不填不得互代。 |

## 十、流程审批规则补充（并行/退回/拒绝）

> 本章为审批类需求的补充口径，来源于已落地用例实践（如“再保理融资新增审批流程”），用于提升流程类测试设计一致性；**不替代**前文规则。

### 10.1 主链路与节点定义

- 先梳理审批主链路，明确每个节点的前后关系、是否存在“首次提交跳过”等特例。
- 每个节点需明确是**串行节点**还是**并行节点**，并在用例中反映节点类型差异。

### 10.2 退回与拒绝统一规则

- **退回规则统一**：若 PRD 指定“默认退回到某固定节点”，则各节点（含并行节点）按该默认节点回退，不得在用例中写成多种回退口径。
- **拒绝规则统一**：任一节点拒绝后流程终止，业务状态为拒绝；若 PRD 允许重发，需补“可再次发起”断言。

### 10.3 并行审批规则

- 并行组内每个节点的“通过”至少拆两类：  
  1) 对侧仍待办时通过；  
  2) 对侧已通过时通过（触发汇聚后进入下一阶段）。
- 并行组内任一侧退回，按 PRD 口径处理（常见为整体退回）；任一侧拒绝，按 PRD 口径处理（常见为整体拒绝终止）。
- 存在两组及以上并行时，需验证“第一组汇聚后进入第二组；第二组汇聚后进入下一串行节点”的阶段衔接。

### 10.4 节点动作覆盖最低要求

- 串行审批节点通常至少覆盖：**通过 / 退回 / 拒绝**。  
- 对存在“首次提交跳过”的节点，需补“首次跳过”与“被退回后再进入该节点”的场景。  
- 并行节点在上述基础上，需增加“对侧状态差异（待办/已通过）”前置。

### 10.5 前置与预期断言写法

- 前置必须写清：当前到达节点、上一节点结论、并行对侧状态、是否首次提交/退回重提。
- 预期至少包含三类断言：  
  1) **流程去向**（进入哪个节点/阶段）；  
  2) **待办变化**（对侧待办是否保留、是否新生成待办）；  
  3) **业务状态**（通过/退回/拒绝后状态值与是否可重发）。

### 10.6 对称性与可复测性

- 并行左右两侧（如财务侧与风控侧）应按**对称规则**设计，避免只写单侧。
- 性能或流程时序类场景（如查询速度优化、汇聚时序）应保留可复测描述（样本规模、触发方式、统计口径）。
